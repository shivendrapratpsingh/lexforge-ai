# LexForge AI — Full Architecture & Connection Map

## Tech Stack Overview

| Layer | Technology | Version | Purpose |
|---|---|---|---|
| Frontend Framework | Next.js (App Router) | 15.2.4 | Full-stack React framework |
| UI | React | 18.3.1 | Component rendering |
| Auth | NextAuth.js v5 beta | 5.0.0-beta.25 | JWT session management |
| Database ORM | Prisma | 5.22.0 | PostgreSQL client |
| Database | Neon (PostgreSQL) | serverless | Stores users + drafts |
| AI | Groq SDK | 0.5.0 | Llama 3.3 70B inference |
| Password | bcryptjs | 2.4.3 | Password hashing (pure JS) |
| PDF Export | jsPDF | 2.5.1 | Server-side PDF generation |
| DOCX Export | docx | 8.5.0 | Server-side Word generation |
| Styles | Tailwind CSS | v4 | Utility CSS (landing page) |

## Authentication Flow (JWT, no database sessions)

```
User submits login form
        ↓
next-auth/react: signIn('credentials', { email, password })
        ↓
POST /api/auth/signin  (handled by [...nextauth]/route.js)
        ↓
lib/auth.js: Credentials.authorize()
    ├── lazy import: lib/prisma.js
    ├── prisma.user.findUnique({ email })
    ├── bcrypt.compare(password, user.password)
    └── returns { id, email, name } or null
        ↓
NextAuth creates JWT token  (signed with AUTH_SECRET)
    └── jwt callback: adds token.id = user.id
        ↓
JWT stored in httpOnly cookie (no database, no sessions table)
        ↓
Every page request: middleware.js reads JWT → authorized() callback
    ├── not logged in + protected route → redirect /login
    └── logged in + auth page → redirect /dashboard
        ↓
Server Components: auth() reads JWT → session.user.id available
```

## Document Generation Flow

```
User fills form → clicks "Generate with AI"
        ↓
POST /api/drafts  { documentType, templateData }
        ↓
auth() → verify JWT session → get userId
        ↓
lib/utils.js: searchCaseLaws(templateData values) → top 3 case laws
        ↓
lib/groq.js: generateLegalDocument(documentType, details)
    ├── Groq API → llama-3.3-70b-versatile
    ├── system prompt: Indian legal expert instructions per doc type
    └── returns: full document text (up to 3000 tokens)
        ↓
prisma.draft.create({ userId, title, content, documentType,
                      templateData, caseLaws, legalReasoning })
        ↓
returns: draft.id
        ↓
router.push('/drafts/[id]')  → view generated document
        ↓
Export buttons: GET /api/export/[id]/[pdf|docx|txt]
    ├── PDF:  jsPDF → A4 layout with title, meta, content
    ├── DOCX: docx library → structured Word document
    └── TXT:  plain text with header/footer
```

## Database Schema Relationships

```
User
  id        (cuid, PK)
  name      String?
  email     String  @unique
  password  String? (bcrypt hash, 12 rounds)
  drafts    Draft[]

Draft
  id             (cuid, PK)
  userId         String  → FK: User.id
  title          String  (auto-generated: "DocType: KeyDetail")
  content        String  @db.Text (AI-generated document)
  documentType   String  (LEGAL_NOTICE | CASE_BRIEF | CONTRACT | PETITION | MEMORANDUM)
  status         String  @default("draft") | "finalized"
  templateData   Json    (the form fields the user filled in)
  caseLaws       Json    (array of { name, citation, summary } objects)
  legalReasoning String? @db.Text
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt
```

## Environment Variables Reference

| Variable | Used By | What It Does |
|---|---|---|
| `DATABASE_URL` | `lib/prisma.js`, `prisma/schema.prisma` | Neon PostgreSQL connection string |
| `AUTH_SECRET` | `auth.config.js` | Signs/verifies JWT tokens. Without it, auth breaks |
| `NEXTAUTH_URL` | NextAuth internally | Base URL for redirect callbacks |
| `AUTH_URL` | NextAuth v5 beta | Same as NEXTAUTH_URL (v5 uses AUTH_URL) |
| `GROQ_API_KEY` | `lib/groq.js` | Groq API key for Llama 3.3 70B |

## Key Design Decisions & Why

| Decision | Reason |
|---|---|
| **No PrismaAdapter** | PrismaAdapter is incompatible with JWT+Credentials strategy in NextAuth v5 — it requires database sessions, which we don't use |
| **Split auth: auth.config.js + lib/auth.js** | `middleware.js` runs in Edge Runtime (no Node.js). `auth.config.js` is Edge-safe (no Prisma/bcrypt). `lib/auth.js` has the full implementation for API routes |
| **bcryptjs, not bcrypt** | `bcrypt` uses native C++ bindings (can't run in some environments). `bcryptjs` is pure JS — works everywhere, webpack can bundle it |
| **Lazy Prisma imports** | `await import('@/lib/prisma')` inside functions prevents app crash if Prisma client wasn't generated yet |
| **Static bcrypt import** | `import bcrypt from 'bcryptjs'` — dynamic CJS `.default` imports return `undefined` in Turbopack. Static import works correctly |
| **Prisma 5.22.0, not 7.x** | Prisma 7 removed `url` from datasource in schema.prisma (breaking change). Prisma 5 uses the standard format we have |
| **JWT, not database sessions** | Simpler — no session table needed, works with Credentials provider, stateless |
| **Groq over OpenAI** | Free tier available, fastest inference globally (Llama 3.3 70B in ~5-10s), no credit card needed |
